1. Get the application URL by running these commands:
  export POD_NAME=$(kubectl get pods -l "app.kubernetes.io/name=autogenstudio,app.kubernetes.io/instance={{ .Release.Name }}" -o jsonpath="{.items[0].metadata.name}")
  kubectl port-forward $POD_NAME 8081:8081
  echo "Visit http://127.0.0.1:8081 to use the application"

2. To view the logs:
  kubectl logs -f $POD_NAME

3. For production deployments, consider:
   - Setting up an Ingress controller
   - Configuring a proper database (PostgreSQL/MySQL) instead of SQLite
   - Adjusting resource limits based on expected load
   - Enabling autoscaling if needed
